# Lessons

- When validating Electron auto-updates on Windows, check `build.win.publisherName` before debugging downloader errors. Leaving it set causes unsigned installers to fail signature validation even if `latest.yml` and asset names are correct.
- After changing Electron window chrome, verify all fixed shell elements against the new title bar height and confirm the main content column still has `flex-1 min-w-0`; otherwise the app can render with overlap and a false empty gutter on the right.
- For Chromium scrollbar theming in the renderer, target `*::-webkit-scrollbar*` and use transparent borders plus `background-clip: padding-box` on the thumb, or Windows will still render square-looking bars even when `border-radius` is set.
- Keep launcher data refresh state separate from presentation ordering. Tying a focus or install refresh key to home-page game ordering causes the catalogue to reshuffle unexpectedly every time the window regains focus.
- If Electron on Windows still ignores rounded scrollbar CSS, disable the Chromium `OverlayScrollbar` feature at process startup before chasing renderer CSS. The website selectors can be correct and still not render as expected until overlay scrollbars are off.
- If the launcher still shows square scrollbars after the native CSS and Chromium flags are corrected, stop relying on native scrollbar painting and move the visible shell containers to a real custom scroll area component. That is the first reliable fix on Windows, not a last resort.
- When converting a native overflow container to a custom scroll area, also convert every high-visibility popup or dropdown that still uses `overflow-y-auto`; otherwise the shell looks fixed but the most obvious overlays still render mismatched native scrollbars.
- When adding a new install path like manual archive extraction, seed it into the shared downloads store immediately and keep secondary activity cards actionable. Modal-local progress and cancel-only secondary cards create dead ends the user cannot recover from.
- Do not make extraction cancel destructive by default. Hydra's archive-management split is the safer model: cancel should stop work and preserve the archive, while archive deletion should be an explicit user action after extraction succeeds.
- Keep pause semantics explicit. If a user pauses an active download, do not silently advance the queue or auto-resume other paused items later. That scheduler behavior reads like the app ignored the pause request.
- When adding archive-deletion prompts or post-install behavior, check ALL download completion paths. UC.Files has its own parallel download engine with a separate `handleUCFilesDownloadComplete` handler that bypasses the normal Chromium download done handler. Changes to the normal path do not automatically propagate to UC.Files.
- For lightbox upgrades, include drag/pan behavior with zoom from the start. Verify the same fix lands in every repo path the user requests, not just one implementation.
- When mirror domains are download-only, do not rely on renderer pages alone. Reject auth in the Electron main-process bridge too, or secondary login buttons will still open broken mirror auth flows.
